Git命令学习笔记

#GIT学习笔记:
Git 的windows版本下载地址：http://msysgit.github.io/

github 地址：http://www.github.com





配置个人信息:
git config –global user.name “NAME” #告诉git 你的昵称

git config –global user.email “email” #告诉git你的邮箱

–global 指定的昵称和邮箱是针对本地所有的 git仓库,丌加该参数时，只是针对当前仓库

配置git显示的颜色:git config –global color.ui ture

在本地创建.gitignore文件可以在里面写上丌需要提交的文件

配置命令的别名:git config –global alias.别名 “命令”
也可以在用户目录下的.gitconfig 里配置别名

ssh-keygen -t rsa -C “80745941@qq.com” 生成公共密钥

ssh -T git@github.com 验证ssh

git remote add origin git@github.com:racall/racall.github.io.git 把当前本地仓库不
远程仓库关联

git push -u origin master 第一推送到远程仓库,以后推送不加参数-u

git clone 远程仓库地址 克隆远程仓库到本地

git init #将当前目录指定为Git仓库

git add #告诉git把文件添加到仓库(实际上是把文件修改添加到暂存区)

git commit -m “commit” #告诉git把文件提交到仓库,引号中的内容为本次提交的说明(把暂存区的内容提交到当前分支)

git status #查看当前仓库状态

git diff #对比某个文件这次与上次的不同

git diff HEAD – #对比某个文件在工作区与版本库中的不同

git log #查看我们提交的历史记录

git reset –hard HEAD^ #回退到上一个版本

git reset –hard ID #回退到指定commit id的版本,commit id一般只写前几(6~7)位

git reflog #查看操作历史记录

git checkout – readme.txt意思就是，把readme.txt文件在工作区的修改全部撤销，这里有两种情况：

一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；

一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

总之，就是让这个文件回到最近一次git commit或git add时的状态。

删除本地文件之后，git rm 可以告诉git你需要从版本库中删除,并且git commit提交。

文件被误删除，可以使用git checkout – 撤销删除

分支:

查看分支:git branch

创建分支:git branch

切换分支:git checkout

创建并切换分支:git checkout -b

合并分支到当前分支:git merge

删除分支:git branch -d

没有合并时，强行删除分支:git branch -D

删除远程分支:git push origin :

关联本地与远程分支：git branch –set-upstream-to=origin/ 或者 git branch –set-upstream origin/

git log –graph –pretty=oneline –abbrev-commit 查看分支图

git merge –no-ff -m “提交说明” 禁用fast forward模式，并建立新提交，合并分支

git stash 存储当前工作区

git stash list 查看存储的工作区

git stash apply 恢复存储的工作区,apply后可以加指定的工作区

git stash drop 删除存储的工作区

git stash pop 删除并恢复存储的工作区

多人协作：

查看远程库信息，使用git remote -v；

本地新建的分支如果不推送到远程，对其他人就是不可见的；

从本地推送分支，使用git push origin branch-name，如果推送失败，先用git pull抓取远程的新提交；

在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致；

从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。

标签：

查看标签：git tag

创建标签：git tag

对某个提交打上标签:git tag Commit-ID

查看某个标签的详细信息:git show

创建带有说明的标签:git tag -a -m “explain” Commit-ID

删除标签:git tag -d

推送某个标签到远程:git push origin

推送全部标签到远程:git push origin –tags

删除远程标签：git tag -d 然后 git push origin :refs/tags/
