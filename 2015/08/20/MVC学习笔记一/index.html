MVC学习笔记一

2015-08-20

　　最近开始学习PHP的mvc架构，可能大家也知道很多的PHP框架,都是采用MVC架构，比如ThinkPHP、yii。MVC全名是Model View Controller，是模型(model)－视图(view)－控制器(controller)的缩写。


  1. Model（模型）表示应用程序核心（比如数据库记录列表）。

  2. View（视图）显示数据（数据库记录）。

  3. Controller（控制器）处理输入（写入数据库记录）。


    4. 　　其实根据我肤浅的理解，Model（模型）就把是从数据库中把数据取出来，View（视图）就是把模型从数据库中取出的数据显示出来，Controller（控制器）就相当于一个遥控器，能让用户操作，并且处理用户输入的内容。这些都是我学习这两天对MVC概念的理解，可能理解的很肤浅,希望大家多多指教！

    下面展示一下我的第一个MVC代码：

    testModel.class.php


    <?php
    /**

    • 模型的作用是获取数据并处理返回数据
      */
      class testModel
      {

      function get()
      {

      return "Hello World";

      }
      }
      ?>

    testView.class.php


    <?php
    /**

    • 视图的作用是将取得的数据进行组织、美化等，并向用户输出
      */
      class testView
      {

      function display($date)
      {

      echo $date;

      }
      }
      ?>

    testController.class.php


    <?php
    /**

    • 控制器的作用是调用模型，并调用视图，将模型产生的数据传递给试图，并让相关视图去显示
      */
      class testController
      {

      function show()
      {

      $testModel=M('test');
$data=$testModel->get();
$testView=V('test');
$testView->display($data);

      }
      }
      ?>

    test.php


    <?php
        require ‘testController.class.php’;
        require ‘testModel.class.php’;
        require ‘testView.class.php’;
        $testController=new testController();
        $testController->show();
    ?>
